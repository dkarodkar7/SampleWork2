Alerts

Usage

  • Limit to 1-2 sentences.
  • Be sure to apply the correct color for an alert’s level.

Levels

We have four levels of alerts:

  • Error: an application error has occurred.
  • Warning: the user is about to perform an irreversible or destructive action; or the result of an action is unexpected, but is not an error.
  • Success: confirmation that the application was able to complete a user’s action or request.
  • Info: the application has non-critical — but helpful — information to share.

Writing Alert Copy

When writing your alerts, focus on providing a clear, simple description of what has happened or what needs to happen next. Here are some basic guidelines for how to write these critical pieces of user interface copy.

Goals

  • Help the user understand what action caused the alert.
  • Explain what the user can do next to remedy the situation, if applicable.

Style

  • Use active rather than passive writing.
  • Focus on what the user most needs to know.
  • Provide solutions to a problem where possible.
  • Construct short sentences, written in plain language.

Capitalization

Alert labels use sentence capitalization, except when they include words or phrases that are acronyms or proper nouns (e.g., “RMA number,” “Cost Account,” “ZIP Code”).

Punctuation

  • Conclude your alert copy with a period.
  • Don’t phrase your alert copy as a question.
  • You can include exclamation points in success alerts. For warning alerts, reserve their use when alerting users to critical information, not to everyday alerts. Never include an exclamation point in an error alert.
