AngularJS Translation

Angular Translate is an AngularJS module (angular-translate) that helps speed tasks related to internationalization (i18n) and localization (l10n) of your product’s UI elements. It supports lazy loading and provides components like filters and directives, asynchronous loading of i18n data and much more. It’s flexible too – you can build your own loaders, storages or error handlers and extend angular-translate to meet your needs.

The angular-translate Getting Started guide provides detailed information on localizing your application. Or contact the Globalization Team at globalization@pb.com or through Yammer for more information.

How It Works

  1. You develop your application in a native language (English, for example) using standard JSON files with keys for each string. (An sample of the JSON is shown after the example below.) The JSON keys are used in views and controllers, instructing angular-translate which string to display.
  2. The Globalization Team consumes the native language JSON files in an application called Passolo and provides JSON files for each translated language.
  3. As you update your native language files, Passolo is used to track what’s changed between versions. This allows translators to focus on what’s new, which saves them time and saves you money.

File Naming

Your JSON file names should be constructed using standard two-letter codes for native language plus country. For example:

  • English (U.S.): en-us.json
  • French (France): fr-fr.json
  • Portuguese (Brazil): pt-br.json

Using Locales

Angular separates number, date and time, and currency formats into different files for each locale, and provides filters that use locale information to format and display this content correctly. A special library, angular-dynamic-locale, allows you to select your locale (and store or retrieve it from a cookie). To take advantage of this functionality, you’ll need to include the angular-dynamic-locale library in your application.

See a list of supported locales or get more information on Angular’s support for i18n and l10n.
